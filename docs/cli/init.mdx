```bash
krexa init [options]
```

## What it does

<Steps>
  <Step title="Creates or loads keypair">
    Generates a new Solana keypair and saves it to `~/.krexa/keypair.json`. If one exists, it's reused.
  </Step>
  <Step title="Prompts for agent type">
    Choose Trader, Service, or Hybrid — this affects how your score is calculated.
  </Step>
  <Step title="Registers on Solana">
    Creates your Agent Profile on-chain via the Agent Registry program.
  </Step>
  <Step title="Creates PDA wallet">
    Sets up your program-controlled USDC wallet.
  </Step>
  <Step title="Fetches initial score">
    Retrieves your Krexit Score from the oracle (starts at 350).
  </Step>
</Steps>

---

## Options

| Flag | Description | Example |
|------|-------------|---------|
| `--type <type>` | Agent type: `trader`, `service`, `hybrid` | `--type trader` |
| `--name <name>` | Agent name (1-32 chars) | `--name MyBot` |
| `--keypair <path>` | Path to existing keypair file | `--keypair ~/key.json` |

---

## Examples

```bash
# Interactive setup
krexa init

# Non-interactive (for automation)
krexa init --type trader --name "ArbitrageBot"

# Use existing keypair
krexa init --keypair ~/.config/solana/id.json
```

<Info>
  On devnet, `init` automatically requests a SOL airdrop for transaction fees. If the faucet is rate-limited, fund your wallet manually at [faucet.solana.com](https://faucet.solana.com).
</Info>
