## Getting Started

Connect Krexa to Cursor so you can borrow, repay, and manage your agent wallet directly from the editor.

<Steps>
  <Step title="Create the config file">
    Add a `.cursor/mcp.json` file in your **project root** with the following content:

    ```json .cursor/mcp.json
    {
      "mcpServers": {
        "krexa": {
          "command": "npx",
          "args": ["-y", "@krexa/cli", "mcp"],
          "env": {
            "KREXA_NETWORK": "devnet"
          }
        }
      }
    }
    ```

    <Note>
      The file must be at `.cursor/mcp.json` inside your project root, **not** in your home directory.
    </Note>
  </Step>

  <Step title="Restart Cursor">
    Restart Cursor completely (or reload the window) so the MCP server is detected.
  </Step>

  <Step title="Verify the connection">
    Open the Cursor chat and ask:

    ```text
    "What's my Krexa wallet balance?"
    ```

    Cursor should detect and call the Krexa `check_balance` tool.

    <Check>
      If you see a USDC balance returned, the integration is working.
    </Check>
  </Step>
</Steps>

<Warning>
  Make sure you've run `npx @krexa/cli init` first to create your agent keypair and register on Solana.
</Warning>

## Customizing the Configuration

You can pass additional environment variables to control network and keypair settings:

```json .cursor/mcp.json
{
  "mcpServers": {
    "krexa": {
      "command": "npx",
      "args": ["-y", "@krexa/cli", "mcp"],
      "env": {
        "KREXA_NETWORK": "devnet",
        "KREXA_KEYPAIR": "~/.krexa/keypair.json",
        "KREXA_RPC_URL": "https://your-rpc-provider.com"
      }
    }
  }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="MCP server not detected" icon="circle-exclamation">
    - Confirm `.cursor/mcp.json` is in your **project root** (not your home directory)
    - Restart Cursor completely (not just a window reload)
    - Verify the server works standalone: run `npx @krexa/cli mcp` in your terminal
  </Accordion>

  <Accordion title="Tools not showing up" icon="eye-slash">
    - Run `npx @krexa/cli init` to set up your agent first
    - Check that `~/.krexa/keypair.json` exists
    - Try setting `KREXA_KEYPAIR` explicitly in the `env` config
  </Accordion>

  <Accordion title="Connection timeouts" icon="clock">
    - Devnet RPC can be slow during peak hours
    - Set a custom `KREXA_RPC_URL` pointing to a dedicated RPC provider for better reliability
    - Check your network connection and firewall rules
  </Accordion>
</AccordionGroup>
