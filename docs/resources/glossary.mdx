---
title: Glossary
description: Key terms used in Krexa documentation.
---

| Term | Definition |
|------|-----------|
| **Agent** | Autonomous software entity that borrows, operates, and earns on Solana |
| **Agent Profile** | On-chain PDA storing agent identity, type, and metadata |
| **BPS** | Basis points. 1 BPS = 0.01%. 10,000 BPS = 100% |
| **Credit Level** | L1-L4 tier determining max credit and interest rate |
| **CPI** | Cross-Program Invocation — one Solana program calling another |
| **Keeper** | Off-chain bot that monitors agent health and triggers liquidations |
| **Krexit Score** | 200-850 credit score for agents, stored on-chain |
| **NAV** | Net Asset Value — wallet value divided by original credit |
| **Oracle** | Off-chain service that co-signs credit requests and computes scores |
| **PDA** | Program Derived Address — Solana account with no private key |
| **Revenue Router** | Smart contract that splits incoming payments for debt service |
| **Senior Tranche** | Lowest-risk LP position, 10% APR, paid first, loses last |
| **Mezzanine Tranche** | Mid-risk LP position, 12% APR |
| **Junior Tranche** | Highest-risk position, 20% APR, absorbs losses first |
| **Type A** | Trader agent — NAV monitored, liquidation on drawdown |
| **Type B** | Service agent — revenue velocity monitored, wind-down on inactivity |
| **Type C** | Hybrid agent — dual enforcement |
| **Wind-down** | Service agent equivalent of liquidation |
| **x402** | HTTP payment protocol for machine-to-machine transactions |
