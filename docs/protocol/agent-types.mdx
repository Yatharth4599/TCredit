# Agent Types

Krexa supports three agent types, each designed for different use cases. Your agent type is set during initialization and determines how the protocol evaluates your activity.

## Types

### Trader (Type A)

Trader agents operate on decentralized exchanges (DEXes). They borrow USDC to execute trading strategies such as arbitrage, market making, or trend following.

- Revenue source: DEX trading profits
- Activity tracked: Trades on whitelisted venues
- Score evaluation: Profitability based on PnL

```bash
npx krexa init --type trader
```

### Service (Type B)

Service agents provide APIs, tools, or AI services. They receive payments through the x402 payment protocol and can define service plans with milestone-based billing.

- Revenue source: x402 API payments
- Activity tracked: Service delivery, milestone completion
- Score evaluation: Revenue generation and service reliability

```bash
npx krexa init --type service
```

### Hybrid

Hybrid agents combine trading and service operations. The protocol tracks both revenue streams and evaluates both activity types.

- Revenue source: DEX profits + x402 payments
- Activity tracked: Both trading and service activity
- Score evaluation: Combined metrics

```bash
npx krexa init --type hybrid
```

## How Agent Type Affects Scoring

The Krexit Score's **Profitability** and **Behavioral** components evaluate agents differently based on type:

| Component | Trader | Service |
|-----------|--------|---------|
| Profitability | Trading PnL | Service revenue margin |
| Behavioral | Trading patterns, venue usage | Service uptime, delivery consistency |

Hybrid agents are evaluated on both dimensions.

## Changing Agent Type

Agent type is set at initialization. To change your agent type, you would need to re-register with a new agent profile. Contact the Krexa team if you need to migrate an existing profile.
