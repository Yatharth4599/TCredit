---
title: Revenue Router
description: How the Payment Router extracts debt service automatically.
---

The Revenue Router is implemented in the Credit Router Solana program. It intercepts all agent revenue and extracts debt service before funds reach the agent.

## Waterfall

```
Incoming payment: $1.00

1. Protocol fee (10%):     $0.10  -> Treasury
2. Senior yield:           $0.04  -> Senior tranche LPs
3. Mezzanine yield:        $0.02  -> Mezzanine tranche LPs
4. Junior yield:           $0.02  -> Junior tranche
5. Debt service:           $0.32  -> Reduces outstanding balance
6. Agent receives:         $0.50  -> Agent's PDA wallet
```

The agent never touches steps 1-5. Funds are split at the Solana program level.

## Why it works

PDA wallets have no private key. The agent cannot bypass the Revenue Router. Every inbound transfer is processed by the Credit Router program, which enforces the waterfall before crediting the agent's balance.

## For traders

Revenue = trading profits. When trades settle through whitelisted venues, profit is captured and routed.

## For service agents

Revenue = x402 payments. When customers pay the agent's endpoint, the payment lands in the PDA wallet and triggers the Revenue Router.

## Surplus distribution

When an agent has no outstanding debt, the waterfall simplifies:
1. Protocol fee (10%) -> Treasury
2. Remainder -> Agent's PDA wallet
